Detailed Description

A number of filters that executes sequentially.

Request handler is executed after the last filter. Each filter can stop filter chain execution by not calling 'filter_chain.do_filter(...)'. In that case filter itself should handle the request. Filter must not write anything to response if it won't going to stop filter chain.
